单词(words)

Description

给你两个由小写字母组成的单词AAA和BBB，我们称一个单词为幸运单词，当且仅当它是由AAA的某个非 空前缀和BBB的某个非空后缀拼接而成的（AAA的前缀在BBB的后缀的前面）。

例如，当单词AAA为treetreetree，单词BBB为heapheapheap时，traptraptrap就是一个幸运单词，而traep,aptrtraep,aptrtraep,aptr则不是。 请问对于给定的单词AAA和BBB，共有多少个不同的幸运单词？

Input

输入包含两行，第一行为单词AAA，第二行为单词BBB。

Output

输出一个整数，表示幸运单词的总个数。

Examples

Input Output
tree
heap		 14

数据范围

对于20%20\%20%的数据，单词AAA的长度lengthAlength_AlengthA​和单词B的长的lengthBlength_BlengthB​满足，1≤lengthA,lengthB≤100;1\le length_A, length_B\le 100;1≤lengthA​,lengthB​≤100;

对于40%40\%40%的数据， 1≤lengthA,lengthB≤103;1\le length_A, length_B\le 10^3;1≤lengthA​,lengthB​≤103;

对于100%100\%100%的数据，1≤lengthA,lengthB≤105;1\le length_A, length_B\le 10^5;1≤lengthA​,lengthB​≤105;

其实答案因该和lengthA×lengthBlength_A\times length_BlengthA​×lengthB​有关

但是看treaptreaptreap， 他可以看成tre+aptre+aptre+ap和tr+eaptr+eaptr+eap

就是说如果直接按照上面直接乘的话，treaptreaptreap是被计算了两次的

所以可以一记录一下A，B串中除了首位字符，有多少个重复的字符，就可以了

所以第一想法就是贪心，这是毫无疑问的

但是考虑到后面的鼹鼠用剩余的点来贪心的话显然是不正确的

怎么办呐?

于是就可以借用网络流的反悔机制

这是111(黑)吃了上面的111的情况，显然如果222要吃饭了的话，111时占据了222的坑位的

所以其实222其实应该吃的是111上面的点，但是我们看见的222现在能吃的只是111下面的点了

让如果我们让这个时候的222经过111吃的点的时候的边权为−1-1−1就等价于111吃的是下面的点，而222吃的是111上面的点了

就是让后来的鼹鼠按相反的方向经过了之前的鼹鼠经过的边时，加上−1-1−1的边权即可
