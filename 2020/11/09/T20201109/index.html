T2(时代的眼泪)

首先交换求和顺序，然后就可以发现诸多有用的性质。

算法一 (25分)

就是每次暴力查询，单次查询的时间复杂度为 O(nlog⁡n)O(n\log n)O(nlogn)。

就是可以用树状数组维护，这一条链有多少个点的权值严格大于当前点，然后就可以加入贡献了。

Code

算法二 (20分)

整张图是一个 1∼n1\sim n1∼n 的链，所以每个点的贡献是一定的，就可以 O(nlog⁡n)O(n\log n)O(nlogn) 预处理出所有点贡献，做两次前缀和就可以了。

结合算法一，就可以获得 454545 分的好成绩。

Code

算法三 (20分)

就是一个以 111 为中心的一个菊花图，这个手摸一下就可以发现，我们可以 O(n)O(n)O(n)，预处理出 111 作为根时的答案，然后在考虑根变成其他的时候的答案变化量，这个变化量的处理单词时间复杂度是 O(log⁡n)O(\log n)O(logn)。

结合算法一、算法二就可以取得 656565 分的好成绩。

About 100

这其实是一个换根 dpdpdp，看上去也很像一个换根 dpdpdp，可惜当时就是没有推出来。

就是考虑，从一个根到另一个根，它少了些什么，它又该多一些什么。

⇓

可以发现，少的那一部分是现在的根对原来的根的贡献，而多的那一部分则是原来的根作为子树对现在的根的贡献。

首先，声明几个数组

  • fif_ifi​ 表示在以 iii 为根的子树中，有多少个点的点权严格小于当前节点
  • gig_igi​ 表示在以 iii 为根的子树中，有多少个点的点权严格小于当前节点的父亲
  • kik_iki​ 表示在所有的点里面，有多少个点的点权严格小于当前节点

所有可以有如下转移：

ansv=ansu−gv+kv−fv;ans_v = ans_u - g_v + k_v - f_v; ansv​=ansu​−gv​+kv​−fv​;

然后 f、gf、gf、g 的求法是十分的巧妙，它可以类似查分的形式出现，所以这道题就结束了。
